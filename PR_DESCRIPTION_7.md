## fix(core): 解除仓库硬编码限制，支持全网任意公开 PR 动态评审

### 【功能描述】
修复了系统此前只能固定评审 `yangshuai128/ai-pr-reviewer` 仓库 PR 的底层 Bug。通过彻底重构 URL 解析与 API 通信模块，系统现在具备了"全网通行"的能力，能够动态接管并分析全网任意公开 GitHub 仓库的合法 PR 链接。

### 【实现思路】
- **动态正则解析 (`src/github_client.py`):** 彻底重构了 `parse_pr_url` 函数，剥离了对固定组织和仓库名的硬编码，改为使用稳健的正则/字符串分割逻辑，精准提取 `owner`, `repo` 与 `pr_number` 三元组。
- **URL 动态挂载:** 审查并重构了 `fetch_pr`、`post_pr_comment`、`post_inline_review` 等所有 GitHub API 请求，将原本写死的 URL 路径替换为基于提取变量的动态 f-string（如 `repos/{owner}/{repo}/pulls...`）。
- **前端链路打通 (`app.py`):** 验证了在多页面架构的【评审中心】中，外部自定义链接能够平滑穿透至后端，并在评论发布环节保持上下文一致性。

### 【测试方式】
1. **外部仓库测试：** 在前端【评审中心】输入 `https://github.com/PyGithub/PyGithub/pull/3294` 并点击开始评审，系统能够成功拉取代码 diff、执行 Claude + ruff 双核检查，并正常渲染多页面看板。
2. **本仓库回归测试：** 重新输入 `本仓库 PR #1` 进行回归测试，功能依旧完好，未引发任何向后兼容性问题。

### 【合并后状态】
- 主分支核心通信链路彻底解耦。
- 产品从"个人单仓库玩具"正式升级为"通用型 SaaS 评审工具"。
